## Harbor with HTTPS: 


### Architecture: 
- Harbor Server: `10.1.1.21`
- Domain: `reg.technbd.com` --> `10.1.1.21` 
- Docker Client (Host) : `10.1.1.190`



## Generate Certificate for Harbor Server:

### 1. Create a CA:


```
mkdir -p /data/cert
cd /data/cert
```


```
-- Generate a CA certificate private key:
openssl genrsa -out ca.key 4096


-- Generate CA certificate: 
openssl req -x509 -new -nodes -sha512 -days 3650 \
  -subj "/C=BD/ST=Dhaka/L=Dhaka/O=Technbd/OU=Personal/CN=Technbd Harbor CA" \
  -key ca.key \
  -out ca.crt
```





### 2. Create the Harbor Server Certificate:


_Generate a private key:_
```
openssl genrsa -out reg.technbd.com.key 4096
```


### 3. Create the CSR:

_Generate a certificate signing request (CSR):_
```
openssl req -sha512 -new \
  -subj "/C=BD/ST=Dhaka/L=Dhaka/O=Technbd/OU=Personal/CN=reg.technbd.com" \
  -key reg.technbd.com.key \
  -out reg.technbd.com.csr
```



### 4. Create the SAN extension (x509 v3) file:

Harbor host that complies with the **Subject Alternative Name (SAN)** and x509 v3 extension requirements.


```
cat > v3.ext <<EOF
authorityKeyIdentifier=keyid,issuer
basicConstraints=CA:FALSE
keyUsage = digitalSignature, nonRepudiation, keyEncipherment, dataEncipherment
extendedKeyUsage = serverAuth

subjectAltName = @alt_names

[alt_names]
DNS.1 = reg.technbd.com
DNS.2 = technbd.com
IP.1 = 10.1.1.21
EOF
```




### 5. Sign the server certificate


_Generate a certificate for Harbor host:_
```
openssl x509 -req -sha512 -days 3650 \
  -CA ca.crt \
  -CAkey ca.key \
  -CAcreateserial \
  -in reg.technbd.com.csr \
  -out reg.technbd.com.crt \
  -extfile v3.ext
```



_List of certificate files:_
```
ll

-rw-r--r-- 1 root root 2049 Jul 13 23:00 ca.crt
-rw------- 1 root root 3268 Jul 13 23:00 ca.key
-rw-r--r-- 1 root root   41 Jul 13 23:01 ca.srl
-rw-r--r-- 1 root root 2151 Jul 13 23:01 reg.technbd.com.crt
-rw-r--r-- 1 root root 1704 Jul 13 23:01 reg.technbd.com.csr
-rw------- 1 root root 3272 Jul 13 23:00 reg.technbd.com.key
-rw-r--r-- 1 root root  274 Jul 13 23:01 v3.ext
```



### 6. Copy certificates for Harbor Server:


_Copy the Server Certificate and Key to the Harbor Directory:_
```
cp reg.technbd.com.crt /data/cert/server.crt

cp reg.technbd.com.key /data/cert/server.key
```



To allow the **Docker client to push and pull images**, the Docker daemon must also trust the certificate so ,convert your server certificate from `.crt` to `.cert`, as the Docker daemon requires this extension.


_Configure the Docker Daemon to Trust the Certificate:_
```
openssl x509 -inform PEM -in reg.technbd.com.crt -out reg.technbd.com.cert
```



### 7. Trust the CA on every Docker client:

Create the Docker certificate directory:
```
mkdir -p /etc/docker/certs.d/reg.technbd.com
```

Copy the `CA certificate` (not `server.crt`) to the Docker client:
```
cp ca.crt /etc/docker/certs.d/reg.technbd.com/


cp reg.technbd.com.cert /etc/docker/certs.d/reg.technbd.com/
cp reg.technbd.com.key /etc/docker/certs.d/reg.technbd.com/
```


_Restart Docker Engine:_
```
systemctl restart docker
```




### 8. Update `harbor.yml`:


_Add/modify the following fields:_
```yaml
hostname: reg.technbd.com


http:
  port: 80

https:
  port: 443
  certificate: /data/cert/server.crt
  private_key: /data/cert/server.key


harbor_admin_password: YourStrongPassword

data_volume: /data

trivy:
  enabled: true
```


_Then apply the configuration:_
```
./prepare


docker compose down -v 


docker compose up -d
```


```
docker ps -a
```




### 9. Verify:

```
openssl x509 -in /data/cert/server.crt -noout -text | grep -A5 "Subject Alternative Name"
```

## Install the Harbor CA certificate on the Docker client Host: 


### 1. Copy the CA certificate to the client:

_From the Docker client:_
```
scp idea@reg.technbd.com:/data/cert/ca.crt /tmp/
```


### 2. Create Docker's certificate directory:

_On the Docker client:_
```
mkdir -p /etc/docker/certs.d/reg.technbd.com

cp /tmp/ca.crt /etc/docker/certs.d/reg.technbd.com/ca.crt
```


```
systemctl restart docker
```


### 3. Verify the certificate:

```
openssl s_client \
  -connect reg.technbd.com:443 \
  -servername reg.technbd.com \
  -CAfile /etc/docker/certs.d/reg.technbd.com/ca.crt
```


_The output should end with:_
```
Verify return code: 0 (ok)
```


### 4. Login: 


```
docker login reg.technbd.com
```
